# Skill创建方法论 - 总览

## 项目介绍

**为什么需要这个方法论？**

传统方式创建技能需要20-30小时，包括：
- 详细调研（2-4小时）
- 完整设计（4-6小时）
- 全新编写（8-12小时）
- 全面测试（4-6小时）

本方法论采用**快速迭代式**，将时间压缩到5-6小时：
- 模板启动（30分钟）
- 复用文档填充（3-4小时）
- 自动验证（30分钟）
- MVP发布（1-2小时）

**时间节省：75%**

## 快速开始

### 3步快速创建技能

**步骤1：评估项目（5分钟）**

```bash
cat checklists/pre-creation-checklist.md
```

填写检查清单，决策是否创建。

**步骤2：创建框架（1分钟）**

```bash
./workflows/create-skill.sh my-skill clawhub
```

框架自动生成：
- SKILL.md主文件
- guides/指导文档
- configs/配置示例
- troubleshooting.md故障排查

**步骤3：填充内容并发布（5小时）**

填充内容 → 自动验证 → 快速发布

## 核心价值

- **快速**：5-6小时完成首次发布（vs传统20-30小时）
- **系统**：完整方法论 + 8个核心文档
- **实用**：模板 + 检查清单 + 脚本
- **自动化**：验证和发布自动化
- **双平台**：ClawHub + GitHub Marketplace

## 适用场景

✅ 推荐：GitHub项目转技能、学习方法论、多平台发布、快速验证想法

❌ 不推荐：非GitHub项目、无文档项目、企业内部私有技能
